# /tradego-review

> 代码/内容修改后的截图确认。符合总裁风格才能上线。
> `/tradego-verify` 通过后才能执行Review。

## 触发时机
- `/tradego-verify` 通过后
- 总裁说"帮我看看改得怎么样"

---

## Review检查清单

### 🎯 页面功能（所有修改都要检查）

```bash
# 用camofox打开页面截图
# 目标URL从最近git变更推断：
# - 新增/修改页面: 检查页面能打开且功能正常
# - 样式修改: 检查布局/字体/颜色是否符合预期

# 检查清单：
# □ 页面标题正确
# □ 主要内容可见
# □ 按钮/链接可点击
# □ 没有明显的排版问题
# □ 图片正常显示
# □ 没有控制台错误（console无红字）
```

### 📝 SEO内容（仅内容修改）

```
□ 文章标题包含目标关键词
□ 文章长度合理（>500字或按产品特点）
□ 产品描述专业准确
□ 图片ALT标签已设置
□ 内链/外链正常
□ 没有明显的语法/拼写错误
```

### 🎨 风格（样式/代码修改）

```
□ 字体/颜色/间距与整体风格一致
□ 移动端显示正常（响应式）
□ 没有引入新的样式问题
□ 符合总裁的设计习惯（参考之前的迭代教训）
```

---

## 执行步骤

### Step 1：打开修改页面

```bash
# 根据git变更确定要检查的页面
# 示例：新增了SEO文章 /products/bolt-hex-head

# 用camofox打开
# 或用browser工具截图
```

### Step 2：截图确认

用 `camofox_snapshot` 或 `browser screenshot` 获取页面截图。

### Step 3：对比检查

根据截图检查上述清单，逐项确认。

### Step 4：输出Review报告

**通过**：
```
## ✅ Review通过

检查页面：<URL>
修改类型：<内容/样式/代码>

功能检查：✅ 通过
SEO检查：✅ 通过（内容修改）
风格检查：✅ 通过

→ 可以交付了。
```

**有问题**：
```
## ⚠️ Review发现问题

检查页面：<URL>

问题列表：
1. ⚠️ 按钮文字过长，被截断
2. ⚠️ 图片ALT标签缺失
3. ❌ 移动端排版错乱

→ 需要修复后再确认。
```

## 简化版（紧急情况）

总裁确认"直接看过了，不用再截图"：

```
## ⚠️ Review简化版

总裁已人工确认内容正确。
跳过截图检查，直接交付。
（注意：事后发现问题的概率较高）
```

---

## Review通过标准

必须全部满足：
1. 页面功能正常（能打开/按钮可点/内容可见）
2. 没有明显的用户体验问题
3. 符合TradeGo的设计风格（专业、外贸B2B）
4. SEO内容质量达标（如果是内容修改）

---

## 交付标准

Review通过后，执行 `/tradego-deploy` 部署。
