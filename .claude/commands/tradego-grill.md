# /tradego-grill

> 执行方案前先追问一轮。强制结构化思考，不漏风险。

## 触发时机
执行以下任务前必须先跑grill：
- 新功能开发
- SEO方案制定
- 重大代码修改
- 新市场开拓决策

也适用于：总裁说"帮我做个xxx"的时候，Agent先grill再执行。

## 追问检查清单

### 🔍 第一轮：需求澄清
```
1. 这个需求的核心目标是什么？（不是解决方案，是问题）
2. 成功的标准是什么？怎么才算"做完了"？
3. 边界在哪里？哪些情况不在范围内？
4. 有没有遗漏的相关方？
```

### ⚠️ 第二轮：风险追问
```
1. 最坏情况是什么？会导致什么问题？
2. 哪些环节最可能出错？
3. 有没有可能导致数据丢失或服务中断的操作？
4. 这次修改会不会影响其他功能？
```

### 🎯 第三轮：替代方案
```
1. 不做这个选择，最差能差到什么程度？
2. 有没有更简单的方案能达到80%的效果？
3. 如果A方案失败了，有B方案吗？
4. 这次决策能回滚吗？
```

### ⏰ 第四轮：执行确认
```
1. 第一步做什么？第二步呢？（分解到可操作）
2. 怎么验证是否成功？
3. 如果中途发现不对，怎么停下来？
4. 完成后要告诉谁？
```

## 使用方法

当总裁提出需求时，先执行：

```markdown
收到任务：<复述任务>

开始追问：

## 🔍 需求澄清
[按清单追问，记录回答]

## ⚠️ 风险追问
[按清单追问，记录回答]

## 🎯 替代方案
[按清单追问，记录回答]

## ⏰ 执行确认
[按清单确认，列出执行计划]

---
结论：[接受执行 / 调整后再执行 / 拒绝（原因）]
```

## 追问原则

- **不嘲讽，不质疑人**：只追问方案，不质疑提出者
- **每个问题都要回答**：不能跳过
- **有不确定性就承认**：不知道 → 承认 → 计划如何验证
- **结论要明确**：接受执行 or 调整后再执行，不能模糊

## 示例对话

总裁：帮我写一个SEO文章

Agent：
```
收到任务：为一个螺栓产品写SEO文章

## 🔍 需求澄清
1. 核心目标 → 提高这个产品在Google的排名
2. 成功标准 → 文章被收录 + 排名进入前20
3. 边界 → 英文市场，津巴布韦买家视角
4. 相关方 → 总裁（最终审稿）

## ⚠️ 风险追问
1. 最坏情况 → 文章质量差，反而影响排名
2. 最可能出错 → 关键词选择错误，内容不符合搜索意图
3. 数据丢失风险 → 无
4. 影响其他功能 → 无

## 🎯 替代方案
1. 不做 → 客户找不到这个产品
2. 更简单方案 → 先生成初稿，总裁再修改
3. 失败方案 → 保存草稿，总裁决定是否重写

## ⏰ 执行确认
1. 步骤 → ①查关键词 ②写大纲 ③写内容 ④检查
2. 验证 → 截图预览给总裁确认
3. 中途停止 → 随时可以ctrl+c
4. 完成后通知 → 发送截图给总裁

---
结论：✅ 接受执行，现在开始第一步
```

---

## 使用场景速查

| 场景 | 是否需要grill |
|---|---|
| 写SEO文章 | ✅ 需要 |
| 修改产品页代码 | ✅ 需要 |
| 部署网站 | ✅ 需要 |
| 客户报价 | ✅ 需要 |
| 紧急hotfix | ⚠️ 简化版（只做第一轮+第四轮） |
| 每天例行检查 | ❌ 不需要 |
| 简单问答 | ❌ 不需要 |
