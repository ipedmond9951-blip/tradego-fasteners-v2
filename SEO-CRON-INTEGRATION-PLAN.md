# SEO Cron 集成方案

## 📊 当前情况

### 现有 SEO 相关 cron
| ID | 名称 | 调度 | 状态 | 错误次数 |
|----|------|------|------|----------|
| `8fc56166` | SEO自动化学习与优化 | `0 9 * * *` | ❌ error | 6 |
| `fa7b7da3` | TradeGo SEO待办工作定期回顾 | `0 11 */2 *` | ❌ error | 4 |

### 现有 cron 的问题
1. **AI 反复调用** - 每次跑要花 60-180s，可能超时
2. **高峰期拥堵** - 9:00 多个 cron 同时跑（SEO + 其他），M3 经常过载
3. **任务重叠** - 9:00 cron 修文章，11:00 cron 评估，重复劳动
4. **缺乏统一入口** - 多个脚本（auto-seo, content-cron, manual）混用

## 🎯 集成方案

### 核心思想
**用新脚本替代旧 agentTurn，AI 只做高价值决策，自动化做执行。**

### 任务分工

| 任务 | 自动化脚本 | AI Agent | 频率 |
|------|------------|----------|------|
| 健康检查 | ✅ | - | 每天 |
| 文章质量审计 | ✅ | - | 每天 |
| 自动优化（格式） | ✅ | - | 每天 |
| 智能优化（内容） | - | ✅ | 每周 |
| 重复检测 | ✅ | - | 每周 |
| 部署 | ✅ | - | 每天 |
| GSC 提交 | ✅ | - | 每天 |
| 学习新规则 | - | ✅ | 每周 |
| 知识沉淀 | ✅ | - | 每天 |

## 🏗️ 新架构

### Level 1: 自动 cron（无人值守）
```
每天 06:00 (避开高峰)
└── seo-universal-cron.sh
    ├── Phase 1: 健康检查 (5 min, 全自动)
    ├── Phase 2: 文章质量审计 (10 min, 全自动)
    ├── Phase 3: 格式自动修复 (15 min, 全自动)
    ├── Phase 4: 重复检测 (5 min, 全自动)
    ├── Phase 5: 构建+部署 (3 min, 全自动)
    └── Phase 6: GSC 提交 (5 min, 全自动)
    
    总时间: ~45 min
    AI 调用: 0 次 ✅
```

### Level 2: 智能 cron（agentTurn 辅助）
```
每周一 09:00 (用新模型，错开高峰)
└── seo-intelligent-optimize
    ├── 读取 seo-universal-cron 日志
    ├── 识别 P0 + 高价值 P1 文章
    ├── 智能生成新内容（5-10 篇）
    ├── 智能添加内链
    └── 学习新规则 → 知识库
    
    AI 调用: 5-10 次
    价值: 100x 于自动 cron
```

### Level 3: 战略 cron（每月）
```
每月 1 号 10:00
└── seo-strategic-review
    ├── 分析 GSC 表现
    ├── 评估 ROI
    ├── 规划下月
    └── 更新技能
```

## 💡 关键设计决策

### 1. 错峰
- 06:00 (无人时段) - 自动化
- 09:00 (人类活跃) - 智能（避开 9:00 拥堵）
- 11:00 双日 - 删除

### 2. AI 节省
- 6 个 SEO 任务从 agentTurn → 1 个 agentTurn
- 每天 AI 调用从 ~10 → 0（自动化）
- 每周 AI 调用从 ~70 → 10

### 3. 价值放大
- 自动 cron 每天跑 154 篇文章
- 智能 cron 每周只处理 5-10 篇高价值
- 战略 cron 每月 1 次全局规划

### 4. 可观测性
- 每天生成 seo-universal-YYYY-MM-DD.log
- 每天生成 seo-universal-YYYY-MM-DD.md 报告
- 失败时通知（via WhatsApp）
